Animations

3 ways:

  • JavaScriptpixel per pixel
  • SVG SMILNot well supported
  • CSS3 Animations
These are the only 3 ways to animations something on a web page. JavaScript (via DOM or Canvas), SVG SMIL (works in Firefox, not sure how good are the other browsers), CSS3 Animations: something new, first implemented in Webkit: create animation in CSS.
animatable.com
  • Based on CSS3 animations (in Firefox Aurora)
  • Supported by webkit
  • Can be degradated in a graceful way (see the demo in Fx4)
  • The layout engine take care of the animation (you just set some key frames)
  • It's CSS!!!
animatable.com

7

History API

This is not the most exciting thing ever, but it's an important new API, that fixes a problem that appeared because of AJAX: broken page history.
  • Manipulating the browser history
github.com
  • Github is full of AJAX
  • One action will change a part of the page, which is "supposed" to break the History (if we click somewhere, we stay on the same page, but the content change)
  • With the history API, we can change the URL and be notified when the user press back/forward: smoother and better experience.
github.com
8

Adaptative layout

A layout that works for any window size doesn't exist. So your layout has to change depending on the size of the screen. It's getting more and more important that now we have mobile devices everywhere.
  • CSS
  • MediaQueries
jasonweaver.name
  • This website works perfectly and looks very good on a PC, a tablet or a phone. Exactly the same markup. Not a different website (not m.jasonweaver.name).
  • The mediaQuery let's you define different layouts for different conditions (handheld, size, orientation, ...)
  • A mediaQuery is dynamic
jasonweaver.name
